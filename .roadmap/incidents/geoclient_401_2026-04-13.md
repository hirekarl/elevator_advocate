# Incident: NYC Geoclient API 401 — 2026-04-13

## Summary

Live testing of the NYC Geoclient v2 API revealed a 401 Access Denied error when `USE_MOCK_GEOCLIENT=False`. The API key in `.env` (`NYC_API_KEY`) is being rejected as invalid or expired by the Azure API Management gateway at `api.nyc.gov`.

## Diagnosis

- **Endpoint**: `https://api.nyc.gov/geo/geoclient/v2/address.json` — correct, unchanged
- **Auth header**: `Ocp-Apim-Subscription-Key` — correct mechanism, confirmed by differing error messages:
  - With key present: "Access denied due to **invalid** subscription key"
  - Without key: "Access denied due to **missing** subscription key"
- **Conclusion**: The key itself is expired or revoked. Code and endpoint are fine.

## Troubleshooting Plan

### Step 1 — Verify key status on the NYC Developer Portal

1. Go to the NYC Developer Portal: https://api-portal.nyc.gov
2. Log in with the account that originally created the subscription
3. Navigate to **Profile → Subscriptions** (or **My Subscriptions**)
4. Check the status of the Geoclient subscription — look for "Active", "Expired", or "Suspended"
5. If expired: renew it. If suspended: check email for any policy violation notices.

### Step 2 — Retrieve a fresh key

1. On the subscription details page, click **Show** or **Regenerate** under the Primary Key field
2. Copy the new key
3. Update `.env` → `NYC_API_KEY=<new_key>`
4. Re-run the live test (see Step 4 below)

### Step 3 — If no existing subscription is found (key was never registered)

1. Register for a new subscription at https://api-portal.nyc.gov
2. Find the **Geoclient v2** API product
3. Subscribe to it — subscriptions are free but require a valid NYC.gov or developer account
4. Copy the generated primary key into `.env`

### Step 4 — Validate the fix

Run this one-liner from the `backend/` directory:

```bash
uv run python -c "
import os
os.environ['NYC_API_KEY'] = 'YOUR_NEW_KEY_HERE'
from services.geoclient import GeoclientService
svc = GeoclientService()
result = svc.get_bin_with_coordinates('1', 'Centre Street', 'manhattan')
print(result)
"
```

Expected: `{'bin': '1001831', 'latitude': '40.712793', 'longitude': '-74.005546'}` (or similar non-empty dict).

### Step 5 — If the API continues to return 401 with a valid key

- Check whether the Geoclient v2 product URL has changed (NYC occasionally migrates API versions)
- Test via curl to rule out Python/requests interference:
  ```bash
  curl -H "Ocp-Apim-Subscription-Key: YOUR_KEY" \
    "https://api.nyc.gov/geo/geoclient/v2/address.json?houseNumber=1&street=Centre+Street&borough=manhattan"
  ```
- Check NYC Open Data / Dev Portal announcements for API deprecation notices
- As a fallback, evaluate the NYC GeoSearch API (no auth required): `https://geosearch.planninglabs.nyc/v2/search?text=1+Centre+Street+Manhattan`

## Resolution (2026-04-13)

Key was regenerated via the NYC Developer Portal (subscription "ElevatorAdvocate",
product "Geoclient V2 User", created 2026-04-12). New key still returned 401 due to
Azure API Management propagation delay on brand-new accounts (estimated 24–48 h).

**Implemented permanent fallback**: `GeoclientService.get_bin_with_coordinates` now
catches 401 responses and transparently delegates to `GeoSearchService`
(NYC Planning Labs, no auth required). When the Geoclient key eventually activates,
it will take precedence automatically — no code change needed.
