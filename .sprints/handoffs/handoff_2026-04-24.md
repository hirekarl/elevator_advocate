### 1. Dual-Window Chronic Filter (Backend)

The core methodology problem was the 30-day LoS window: SODA complaints lag by
weeks/months, so 30-day windows systematically undercounted active problems and
produced unreliable AI summaries.

**New approach — `ConsensusManager.get_chronic_offender_data()`:**
- A building qualifies as a chronic offender if it has **1+ complaints in the last
  12 months AND 3+ complaints in the last 3 years**.
- The 12-month window absorbs SODA filing lag. The 3-year bar excludes buildings
  with stale history but no recent activity.
- Returns `None` for nominal buildings; returns `{"complaints_12mo": n, "complaints_3yr": n}` for chronic ones.

**Supervisor NOMINAL gate updated:**
- Added `complaints_12mo` to the hard NOMINAL pre-check in `orchestration/supervisor.py`.
- Previously: `if los == 0 and not active_crisis` → returned NOMINAL (too aggressive;
  missed buildings with SODA lag).
- Now: `if los == 0 and not active_crisis and not complaints_12mo` → only returns NOMINAL
  when there is genuinely zero recent activity.

**`generate_district_reports` command updated:**
- Non-chronic buildings now get a programmatic NOMINAL summary without calling Gemini.
- Chronic buildings receive the full Supervisor → worker pipeline.
- Passes `complaints_12mo` and `complaints_3yr` in the Gemini context.

**New `export_district_csv` management command:**
- Exports a district data snapshot to CSV.
- Columns: `BIN, Address, Borough, Legal Owner (MapPLUTO), Complaints (12mo), Complaints (3yr), Loss of Service % (30d), Risk Level, AI Headline`.
- Flags: `--district`, `--output`, `--chronic-only`.
- Default path has a `..`-level bug — always use `--output` flag explicitly until fixed.

**`DistrictViewSet.report` updated:**
- Replaced 30-day complaint ranking with `get_chronic_offender_data()`.
- Stats now expose `chronic_offenders` count (was `avg_loss_of_service`).
- Removed misleading `management_company` field.

**`DistrictReport.tsx` frontend fix:**
- The API shape change broke the component. Fixed same session.
- `avg_loss_of_service` → `chronic_offenders`; `complaint_count` → `complaints_12mo`; `management_company` removed from interface.
- i18n key `dr_avg_los_label` → `dr_chronic_offenders_label`.
